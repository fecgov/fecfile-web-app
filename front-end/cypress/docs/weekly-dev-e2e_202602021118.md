# FECFILE-2842: E2E flakiness review

## scope
we're seeing flaky E2E tests: intercepts missing, unexpected redirects, and async timing issues

**files in scope**
- `cypress/e2e-smoke/F3X/setup.ts`
- `cypress/e2e-smoke/F3X/utils/`
- `cypress/e2e-smoke/requests/`
- `cypress/e2e-smoke/pages/`
- `cypress/e2e-smoke/models/`

**goal**
- diagnose flakiness causes
- re-examine tests for our W5 coverage (who/what/when/where/why)
- cypress‑side refactors and test hardening
- will impact the distinction b/w `e2e-smoke` and `e2e-extended`
    - `e2e-smoke`: automate flows for heavy tasks (reports, transasctions, batch input)
    - `e2e-extended`: nitty-gritty, atomic/decision-level app behavior of those same flows

---

## commonalities
- tests wait for network requests that either don’t match the intercept or don’t happen at all
- tests assert on UI choreography (spinners, extra GETs, resolver timing) instead of observable outcomes
- cypress is counting requests and waiting for “the 3rd time” a call happens, which is brittle

---

## findings

1) **session reuse causes redirects**
- `LoginPage.login()` uses `cy.session(..., cacheAcrossSpecs: true)` without `validate`
- front-end guards rely on `ffapi_timeout`; stale cookie redirects to `/login`
- flake symptom: mid‑spec redirects
- files: `loginPage.ts`, `login.service.ts`, `logged-in.guard.ts`

2) **report list intercept is too broad**
- `ReportListPage.goToPage()` intercepts `GET /api/v1/reports/**` and waits
- also matches `/reports/form-3x/coverage_dates/` and `/reports/:id/`
- `cy.wait('@GetReports')` can resolve on the wrong request
- files: `reportListPage.ts`, `report.service.ts`, `views.py`

3) **method‑only intercepts are too broad**
- `cy.intercept({ method: 'Post' })` matches any POST, can satisfy wait too early
- files: `transactionDetailPage.ts`, `loans-bank.cy.ts`

4) **exact querystring matching is brittle**
- front-end builds params with `HttpParams`; back-end doesn’t care about order
- tests expect exact ordering via intercept/alias
- files: `transactionDetailPage.ts`, `loans-individual.cy.ts`, `loans-bank.cy.ts`,
  `transaction-list.service.ts`, `table-list-base.component.ts`, BE `views.py`

5) **DataSetup uses Cypress.Promise.all**
- commands may run outside the Cypress chain; race with intercept/alias registration
- file: `setup.ts` (and one call site in `reattributions.cy.ts`)

6) **fixed `cy.wait` calls**
- `cy.wait(100)` / `cy.wait(1000)` can mask real async and fail on CI
- File: `pageUtils.ts`

7) **hard‑coded API host**
- many intercepts use `http://localhost:8080` while front-end uses `environment.apiUrl`
- breaks in CI / proxy / HTTPS
- files: `methods.ts`, `loginPage.ts`, `pageUtils.ts`, many specs

---

## cross‑checks with front-end/backend behavior

**transactions list**
- front-end calls `/api/v1/transactions/?page=…&ordering=…&report_id=…&schedules=…&page_size=…`
- backend accepts params in any order
- tests assuming fixed order are fragile

**report list**
- front-end list: `/reports/?page=…&ordering=form_type`
- resolver also calls `/reports/:id/`
- broad intercepts match both

**auth**
- redirects depend on `ffapi_timeout` cookie
- cached sessions without validation will expire mid‑suite

## failures mapped to likely causes

### 1) GuarantorList timeouts
**specs:** `loans-committee.cy.ts`, `loans-individual.cy.ts`
**Error:** `cy.wait('@GuarantorList')` timed out (no request occurred)

**likely causes**
- intercept doesn’t match (query ordering, host mismatch)
- request doesn’t happen (UI updates locally w/o refetch)

**promises**
- DELETE succeeds and row disappears
- follow‑up GET is optional so waiting for it is flaky

---

### 2) review report totals stay `$0`
**spec:** `review-report.cy.ts`
**Error:** expected `$200.01` but UI stays `$0`

**root cause**
- summary UI uses `activeReport`
- `ReportDetailedSummaryComponent` only refreshes when `calculation_status` is missing or not `SUCCEEDED`
- if calc is triggered out‑of‑band, UI doesn’t refetch which begets stale values

---

### 3) spinner never appears
**spec:** `review-report.cy.ts`
**Error:** `img.fec-loader-image` not found

**root cause**
- spinner only appears while `calculation_status !== 'SUCCEEDED'`
- if status is already `SUCCEEDED`, spinner never shows
- if calc is fast, spinner may appear and disappear before assertion

---

### 4) `GetLoans` “3rd request” timeout
**spec:** `disbursements.cy.ts`
**Error:** waiting for 3rd `@GetLoans` request

**root cause**
- `ReportListPage.goToReportList()` already sets & waits for those aliases
- test repeats intercepts and waits again and gets stuck in an misadventured 3rd request

## constraints & decisions
- **assume front-end/backend code is correct and works as intended**
- **focus only on cypress refactors**
- **keep/augment important UI assertions**

### a) intercepts
- prefer `pathname` w/ a `query object` or regex that matches key params
- avoid full querystring exact matches
- avoid method‑only intercepts unless intentional

### b) aliased waits
- replace “wait for refetch GET” with:
  - wait for **DELETE/POST** and
  - assert **UI state** (row gone / value updated / toast shown)
- avoid waiting for “Nth identical GET”

### c) spinner assertion
make spinner assertions deterministic by controlling timing in cypress

**delay the report refresh GET**
```ts
const setupSummarySpinnerTiming = (reportId: string) => {
  cy.intercept('POST', '**/web-services/summary/calculate-summary/').as('calcSummary');
  cy.intercept('GET', `**/reports/form-3x/${reportId}/`, (req) => {
    req.continue((res) => res.setDelay(1200));
  }).as('getReport');
};

ReviewReport.Summary();
cy.get('img.fec-loader-image').should('be.visible');
cy.wait('@getReport');
cy.get('img.fec-loader-image').should('not.exist');

```
